## 1. 프로젝트 개요

### 1.1 프로젝트 목적

산업용 디지털 트윈 모니터링 시스템의 클라이언트 중심 고성능 웹 프론트엔드 구현을 통해 다음을 달성합니다:

- 서버 중심 스트리밍 방식의 구조적 한계(고비용 GPU 서버, 확장성 제약, 자원 낭비) 해결
- 클라이언트 측 3D 렌더링 + 모의 실시간 데이터로 서버 부하 최소화
- 무제한 확장성 및 제로 서버 비용(OPEX zero) 달성
- 실제 ROS2/WebSocket 연결 시 재사용 가능한 아키텍처 설계

### 1.2 핵심 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| **Core** | React 18 + TypeScript, Vite 5 | 2026 최신 버전 |
| **3D 렌더링** | @react-three/fiber, drei, three.js | 클라이언트 100% 렌더링 |
| **상태 관리** | Zustand | FSD 구조 최적화 |
| **UI** | Material-UI (MUI v5) | 산업용 디자인 |
| **시뮬레이션** | requestAnimationFrame + lerp/easing | 부드러운 움직임 |
| **에셋 최적화** | glTF/GLB (Draco + Meshopt 압축) | 파일 크기 60~70% ↓ |
| **배포** | Vercel | 정적 호스팅, 글로벌 CDN |

- **Task 8.1: 단일 로봇 집중 모드로의 피벗 (Pivot to Single Robot)**
  - 복잡한 Overview(공장 전체) 모드를 제거하고 단일 로봇 모니터링에 자원 집중
  - 불필요한 카메라 줌 기능을 비활성화하여 안정적인 모니터링 시야 확보
- **Task 8.2: 유기적 조인트 연결 및 자연스러운 움직임 구현**
  - UR10의 실제 기하학적 구조를 반영한 조인트 축(Axis) 정밀 매핑
  - ROS2 `sensor_msgs/JointState` 규격을 따르는 시뮬레이션 데이터 연동
- **Task 8.3: 시각적 폴리싱 (Visual Polishing)**
  - `drei`의 `Outlines`를 활용한 Cyan Glow 효과 적용
  - `Stage` 환경 설정을 통한 조명 및 그림자 최적화
- **상태: 완료 (100%)**
